# Frontend Polish Rules

## Non-negotiables
- Do not rewrite architecture.
- Do not change routing, data flow, backend contracts, or business logic.
- Preserve the current color family and brand palette. You may rebalance tones, contrast, surfaces, shadows, and neutrals, but do not replace the core palette.
- Content may stay the same; design system must improve.

## Design goals
- Make the UI feel premium, intentional, and consistent.
- Fix spacing, alignment, hierarchy, radii, shadows, borders, hover states, empty states, table/list density, and mobile responsiveness.
- Eliminate inconsistent card styles, incorrect colors, weak contrast, cramped areas, and random component sizing.
- Unify all cards, buttons, inputs, badges, tabs, side panels, tables, and section wrappers into one system.

## Implementation rules
- Start with shared tokens and reusable UI primitives first.
- Then update all pages using the new primitives.
- Avoid generic AI-looking design.
- Keep the code clean and maintainable.
- Run build after changes and fix all errors.
- Review every route before stopping.

## Deliverables
- Updated frontend
- Short summary of changes
- List of remaining weak spots, if any
